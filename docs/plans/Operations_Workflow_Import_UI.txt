# Operations_Workflow_Import_UI

## Summary

Add a Settings sub-section named `Operations` for workflow bundle import, validation, staging, and explicit activation. Keep the implementation on the existing backend workflow APIs and avoid adding new durable staged-import management unless a later issue requires list/discard behavior.

## Key Changes

- Extend the desktop Settings section model with `Operations`, placed before `Diagnostics`
- Add an Operations panel that shows current active workflow status from `GET /api/workflow/status`: workflow ID, workflow version, state-machine version, content hash, activation time, and supported hook handlers.
- Add a workflow bundle import form:
  - JSON file input using browser/Tauri web file APIs.
  - Notes field sent as `notes`.
  - `Validate and stage` button that parses JSON client-side, then calls `POST /api/workflow/imports`.
- Render staged import result in-session:
  - staged import ID
  - validation status
  - identity fields
  - warnings and errors
  - content hash
  - clear warning that Memo Capture stores only the active workflow bundle body and rollback requires re-importing a known-good bundle.
- Add explicit activation controls only when the staged result is valid:
  - activation notes field
  - required confirmation checkbox
  - `Activate workflow` button calling `POST /api/workflow/imports/{stagedImportId}/activate` with `confirmActivation: true`.
- After successful activation, refresh workflow status and workflow buckets so the Work queue reflects the new active definition.
- Keep this pass UI-only against existing APIs; do not add staged import list/discard routes.

- `Operations` should be a Settings sub-section because the issue reports it missing from the Settings page.
