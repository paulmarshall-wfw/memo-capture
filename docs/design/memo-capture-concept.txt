The intent is to capture ideas in the form of voice memos (with or without transcriptions) and text memos (which might originate from various types of apps meaning they could be in a variety of file formats)

Users will be instructed to start their memos with the following:
- project name
- feature group name
- feature name
- ... followed by their memo
- attribution

The Settings page for this app should allow for the configuration of:
- valid project names plus keywords which could be used to match memos to each project (the set of project names should be fixed)
- project context (a long form description of the project)
- valid feature group names plus keywords which could be used to match memos to each feature (the set of feature names should be dynamic meaning new feature groups can be added as they arise in new memos or are added by users)
- a list of contributors who are allowed to add memos to the system

Memos could be ingested into the system in any of three ways to start with:

1. A data entry form in the app which captures contributor (drop down list of contributors), project name (via a drop down menu), feature group name (via a drop down menu or text field to add a new feature group), feature name (freeform text field), and memo (freeform text field).
2. A watched folder for ingesting audio files (with or without transcriptions).
3. A watched folder for ingesting text files in various formats. The list of supported file types should be configurable in the settings page.

All fields are mandatory except for the contributor field.

Pre-processing of voice and text memos
- If the voice memo has a transcription, it should be ingested as a text memo using the transcription to populate the project name, feature group name, and feature name and memo fields. If there is no transcription, the voice memo should processed to extract the project name, feature group name, and feature name and memo fields. The contributor might be identified in the text of the transcription or possibly by voice identification
- The text files should be processed to extract the project name, feature group name, and feature name and memo fields.
- Keywords should be extracted from every memo (from whatever source) and stored in a list of keywords per memo. These could be used as tags for the memo. 

Once any required pre-processing has been completed work items should be created for each memo and they should enter the workflow lifecycle in the 'new idea' state

Read the file to understand the contract with the state workflow definition and runtime library: '/Users/paulmarshall/Software Development/state-workflow-runtime/docs/target-apps/README.md'

Users will be able to view work items (in the 'new idea' state) in the 'new ideas' bucket

The workflow definition will describe valid transitions and actions in the workflow.

In the 'new idea' state the user will flesh out the memo in the right hand detail panel. I want a button in the right hand panel (work item detail panel) to allow the user to submit two context fields; the work item together with the project description in a prompt to a large language model. The user should be able to configure the base prompt for the large language model. The base prompt and the two context fields should be submitted to the AI model to return an expansion of the memo plus additional memos/ideas which are related to the original memo. Any new memos generated by the AI model should be added to the 'text document watched folder' for automatic ingestion by the system.

Users will be able to 'accept', 'park' or 'reject' work items. This will be defined in the workflow definition.

When viewing lists of work items the users should be able to browse memos using various sorting and filtering options. 
For example users might filter by project name, feature group name, contributor name, keyword or date range. They might sort by project name, feature group name, contributor name, etc.
The list of work items should include the project name, feature group name, feature name, contributor name, and a snippet of the memo. The Work Item detail panel on the right side of the screen should show the full memo

I want this to be a native app that will run on windows and macOS so that it can be used by different people on their own computers. I want a common back-end database though that is accessible to all users. I want to be able to host the back-end database on a cloud service or on my NAS
